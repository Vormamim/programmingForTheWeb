# Day 5 — Authentication, Authorisation and Big Data

**Syllabus:** Authentication and authorisation; investigate the effect of big data on web architecture — data mining, metadata, streaming service management
**Outcomes:** SE-12-03, SE-12-04
**Textbook:** Sections 11.3 + 11.4

---

## Key Terms

**Authentication** — The process of verifying that a user is who they claim to be. Answers the question: *Who are you?* Example: entering a username and password.

**Authorisation** — The process of determining what an authenticated user is allowed to do. Answers the question: *What are you allowed to do?* Example: a logged-in user can view their own account but not another user's.

**Multi-factor authentication (MFA)** — Requiring two or more forms of verification before granting access. Factors are: something you know (password), something you have (phone), something you are (fingerprint).

**Session** — A temporary connection between a client and server that persists across multiple requests. A session is established after authentication and destroyed when the user logs out or the session expires.

**Session token** — A unique identifier generated by the server after authentication. Stored in the browser (usually as a cookie) and sent with each request to prove the user is still authenticated.

**Cookie** — A small piece of data stored by the browser and sent to the server with each request. Used for session management, personalisation, and tracking.

**Big data** — Extremely large datasets that cannot be processed using traditional database tools. Characterised by the **three Vs**: Volume (large scale), Velocity (generated rapidly), Variety (many different formats).

**Data mining** — The process of analysing large datasets to discover patterns, correlations, and insights that are not immediately obvious.

**Metadata** — Data that describes other data. Example: a photo file's metadata includes the date taken, camera model, GPS location, and file size — not the photo itself.

**Streaming** — Delivering data (audio, video) continuously in real time, rather than downloading the entire file before playback begins.

**Content Delivery Network (CDN)** — A distributed network of servers geographically spread across the world that cache and deliver content to users from the nearest server, reducing latency.

## Theory

### Authentication vs Authorisation

These two concepts are closely related but distinct. Confusing them is a common exam error.

| | Authentication | Authorisation |
|---|---|---|
| Question | Who are you? | What can you do? |
| When | Login | After login, on each action |
| Example | Entering username + password | Admin can delete posts; regular user cannot |
| Failure result | Access denied entirely | Access denied to specific resource |

**Example scenario:** A school learning management system.
- **Authentication:** Students log in with their school email and password
- **Authorisation:** Students can view their own grades, teachers can view all grades, admin can modify grades

### Authentication Factors

Authentication systems use one or more of:

1. **Something you know** — password, PIN, security question
2. **Something you have** — phone (SMS code), hardware key, authenticator app
3. **Something you are** — fingerprint, face recognition, iris scan

Single-factor authentication (password only) is weak — passwords can be guessed, stolen, or phished. Multi-factor authentication significantly reduces the risk of unauthorised access.

### Sessions and Cookies

HTTP is **stateless** — each request is independent and the server has no memory of previous requests. Sessions solve this problem.

```
1. User submits login form (username + password)
2. Server verifies credentials
3. Server creates a session and generates a session token
4. Server sends session token to browser as a cookie
5. Browser stores the cookie
6. Every subsequent request includes the cookie
7. Server reads the token, identifies the user, processes the request
```

**Session security considerations:**
- Session tokens must be long and random (not guessable)
- Sessions should expire after inactivity
- Tokens must be invalidated on logout
- Should only be transmitted over HTTPS

### Big Data

Big data refers to datasets so large and complex that traditional database tools cannot process them efficiently. The **three Vs** define it:

- **Volume** — terabytes or petabytes of data (e.g. every tweet ever sent)
- **Velocity** — data generated at high speed (e.g. real-time stock prices, sensor data)
- **Variety** — structured (tables), unstructured (text, images, video), and semi-structured (JSON, XML)

**Why it matters for web architecture:** traditional relational databases are not designed for this scale. Big data requires distributed systems — data is spread across thousands of servers and processed in parallel.

### Data Mining

Data mining is the extraction of useful patterns from large datasets.

**Examples of data mining in web applications:**
- Netflix analyses viewing history to recommend content
- Amazon identifies purchasing patterns to suggest related products
- Banks detect unusual transaction patterns to flag potential fraud
- Social media platforms identify trending topics in real time

**Common data mining techniques:**
- **Classification** — categorising data into predefined groups (spam vs not spam)
- **Clustering** — grouping similar items without predefined categories (customer segments)
- **Association** — finding items that frequently occur together ("customers who bought X also bought Y")
- **Regression** — predicting a numerical value from existing data (house price prediction)

### Metadata

Metadata is often described as "data about data". It provides context without containing the content itself.

**Examples:**
- An email's metadata: sender, recipient, timestamp, subject, size — not the message body
- A photo's metadata (EXIF data): camera model, shutter speed, GPS coordinates, date taken
- A web server log entry: IP address, timestamp, URL requested, response code, browser used

**Why metadata matters:**
- Metadata can reveal sensitive information even when content is protected (e.g. knowing *who* emailed *whom* and *when* reveals relationships)
- Web server logs are metadata — they record every request without storing the content
- GDPR and privacy laws increasingly regulate metadata collection

### Streaming Service Management

Video streaming platforms (Netflix, YouTube, Spotify) face significant architectural challenges:

**The problem:** delivering high-quality video to millions of simultaneous users with minimal buffering.

**Solutions:**

**CDN (Content Delivery Network):** Content is cached on servers worldwide. When you stream a video, it is served from the nearest CDN node — not from a central server. This reduces latency and distributes load.

**Adaptive bitrate streaming:** The video is encoded at multiple quality levels. The streaming client monitors your connection speed and automatically switches between quality levels to maintain smooth playback.

```
Connection fast:   stream at 4K (high bitrate)
Connection drops:  automatically switch to 1080p
Connection slow:   switch to 720p or 480p
```

**Load balancing:** Incoming requests are distributed across many servers so no single server is overwhelmed.

## Objective Response Questions

**1.** A user logs into a web application with their username and password. The system then checks whether they have permission to access the admin panel. Which terms correctly describe these two steps in order?

- A) Authorisation, then authentication
- B) Authentication, then authorisation
- C) Encryption, then authentication
- D) Hashing, then authorisation

**Answer: B** — Authentication (who are you?) comes first, then authorisation (what can you do?).

---

**2.** A company collects GPS location, device model, and timestamp data from users of its mobile app. No messages or personal content is collected. This data is best described as:

- A) Plaintext
- B) Ciphertext
- C) Metadata
- D) Session data

**Answer: C** — Metadata describes context (location, device, time) rather than content.

---

**3.** A video streaming service automatically reduces video quality when a user's internet connection becomes slow. This feature is an example of:

- A) Content Delivery Network caching
- B) Adaptive bitrate streaming
- C) Load balancing
- D) Data mining

**Answer: B**
